# AiSci Dashboard Integration - Megaprompt

## Context
We have a beautifully styled React + Vite frontend dashboard generated by Lovable, located in `apps/aisci-dashboard`. It uses Tailwind CSS v4, shadcn/ui components, Recharts for visualizations, and TanStack Router for navigation. 
Currently, the dashboard relies entirely on static mock data in `apps/aisci-dashboard/src/lib/mock-data.ts`.

## Goal
Transform the frontend mockup into a fully functional, data-driven dashboard connected to the actual AiSci Python ecosystem. The goal is a production-ready view into the continuous literature intake, physics fits, and agent operations.

## Phase 1: Scaffold the API Backend (FastAPI)
The UI currently expects very specific JSON shapes (defined in `mock-data.ts`). We must build a Python backend that provides this exact shape to minimize frontend churn.

1. **Create the API App**: Create `ignition/api.py` using `FastAPI` (or similar).
2. **Configure CORS**: Allow requests from the Vite dev server (`http://localhost:5173`).
3. **Implement Endpoints**:
   - `GET /api/literature`: Query `evidence_graph.db` (the `papers` table) to return recent literature intake. Map the SQLite rows to the `Paper` type (source, category, title, published, abstract, claims, url).
   - `GET /api/evidence`: Query `evidence_graph.db` (the `claims` and `evidence` tables) to return the status of physics claims. Map to the `EvidenceRow` type.
   - `GET /api/fits`: Parse the CSV artifacts in `research/robert/runs/2026-07-09-jacobian-fix/` (`fit_quality.csv`, `fit_parameters.csv`) and return the exact structure needed for the UI's Fit Table and Charts (the `FitRow` type and `chi2Series`).
   - `GET /api/tasks`: Read the project tasks, perhaps parsed directly from `research/robert/next-actions.md` or a dedicated table.
   - `GET /api/agents`: Expose agent activity logs (can be parsed from system logs or database if tracked).

## Phase 2: Wire the React UI (TanStack Query)
1. **Set up Data Fetching**: `@tanstack/react-query` is already in `package.json`. Set up the `QueryClientProvider` in `src/router.tsx` or `src/routes/__root.tsx`.
2. **Replace Mock Data**:
   - Deprecate `src/lib/mock-data.ts`.
   - Update `src/routes/fits.tsx` to fetch from `/api/fits`.
   - Update `src/routes/literature.tsx` to fetch from `/api/literature`.
   - Update `src/routes/evidence.tsx` to fetch from `/api/evidence`.
   - Update `src/routes/tasks.tsx` and `src/routes/agents.tsx`.
3. **Handle Loading/Error States**: Use shadcn/ui `Skeleton` components while data is loading. Add error boundaries using the provided `error-page.ts`.

## Phase 3: Live Integration & Polish
1. **Run Script**: Add a `start_dashboard.sh` script to launch both the FastAPI backend (e.g., `uvicorn ignition.api:app --reload`) and the Vite frontend (`npm run dev --prefix apps/aisci-dashboard`) simultaneously.
2. **Refine Fit Charts**: Ensure the Recharts components in the Overview page receive dynamic data and that the log-scale $\chi^2/\text{ndf}$ chart correctly highlights the rejection threshold (>5).
3. **Semantic Styling**: Ensure the UI's semantic tokens (emerald/amber/rose) correctly reflect backend thresholds dynamically.

## Phase 4: Actionable Operations (Mutations)
The dashboard should not be strictly read-only. We need to empower the user to manage the project loop directly from the UI.
1. **API Mutations (Python)**:
   - `POST /api/ingest`: Trigger the `ignition/ingest_pipeline.py` script. Support streaming logs back to the frontend.
   - `POST /api/fits/run`: Trigger `deployment/helper/run_a1_jacobian_fits.py` or equivalent fitting scaffolds to regenerate fits.
   - `PATCH /api/evidence/{id}`: Approve or reject evidence, moving claims from 'Proposed' to 'Sanity Checked' or 'Supported'.
   - `PATCH /api/tasks/{id}`: Update task statuses directly from the UI.
2. **Frontend Wiring (`useMutation`)**:
   - Wire the "Run Ingest" button to trigger `POST /api/ingest` and display a Sonner toast with the live status.
   - Add action buttons to the Evidence and Task rows to allow state mutations using `@tanstack/react-query` `useMutation` hooks, followed by query invalidation to refresh the UI immediately.

## Instructions for the Agent
1. **Strictly adhere to the frontend contracts**: Start by reviewing `apps/aisci-dashboard/src/lib/mock-data.ts`. Build your FastAPI models (`pydantic`) to match these structures exactly.
2. **Iterative wiring**: Wire one page at a time (e.g., Literature first, then Fits). Verify it works before moving to the next.
3. **Preserve styling**: Do not alter the Tailwind classes or the glassmorphism aesthetic unless absolutely necessary for rendering dynamic state.
4. **Python Standards**: Follow the project's strict typing and linting rules when building the FastAPI backend.
