# AISCI Webapp — v0 concept draft

This is a **design concept** for [AIS-16 — AISCI Webapp / Multica Module](https://github.com/badmarsh/aisci/issues), generated with [v0](https://v0.app) on 2026-06-01 as a first visual pass before any implementation decisions.

## Live preview

- **v0 chat (durable):** https://v0.app/chat/qKjw6fFTOqP
- **Live demo (rendered by v0, link rotates):** see `latestVersion.demoUrl` in the chat page

The v0 chat is the source of truth for this concept. The files in this directory are a snapshot of what v0 emitted, committed for review only.

## What's in scope here

Both roles and all six surfaces, with role-aware default landing:

**Scientist Robert** (lands on Evidence):
- `evidence-ledger.tsx` — claims table with status pills (Open / Sanity checked / Supported / Refuted), counts, χ²/ndf column
- `fitting-pipeline.tsx` — current `fitting_pipeline.py` run with χ²/ndf sparkline, parameter convergence, stop/restart
- `spectra-plotter.tsx` — HEPData pT-spectrum placeholder with dataset picker
- `symbolic-validation.tsx` — formula extraction list with dimensional-analysis pass/fail badges

**DevOps Marek** (lands on Ops):
- `tests-dashboard.tsx` — pytest pass/fail per subsystem (Physics, Onyx, DeerFlow), coverage strip, performance summary, recent-failures list
- `ops-surface.tsx` — deployment status cards (Onyx, DeerFlow, MCP proxy), Docker stack health strip with GPU indicator, Secret-Leak Scanner pill, log tail

Shared shell:
- `top-bar.tsx` — logo, role switcher, search, avatar
- `left-rail.tsx` — section nav (Evidence / Pipeline / Spectra / Validation / Tests / Ops)

## What's out of scope (intentionally)

- **Backend wiring.** All data is mocked inline. There is no data fetching or auth integration yet.

## Running locally (optional)

`page.tsx` imports four shadcn primitives v0 assumes already exist in the project:
`@/components/ui/button`, `@/components/ui/dropdown-menu`, `@/components/ui/avatar`, and `@/lib/utils`.

To make this buildable locally:

```bash
cd webapp
npx shadcn@latest init        # answer prompts, accept defaults
npx shadcn@latest add button dropdown-menu avatar
pnpm install                  # or npm/yarn
pnpm dev                      # http://localhost:3000
```

If you only want to look at the design, just open the v0 demo link above — no setup needed.

## Agent integration path

Stack matches our intended Next.js 14 App Router, React Server Components, shadcn/ui, Tailwind, lucide-react frontend, so this directory can later be lifted into a primary module by replacing mock data with API-fetched data and wrapping in the auth shell.
