# AiSci Scientific-Workflow Coverage

This document assesses how well the `badmarsh/aisci` repository supports the end-to-end scientific workflow, from question formulation to publication. Each stage is marked as Supported, Partial, or Absent, backed by evidence from the codebase.

## 1. Question Formulation
**Status:** Absent / Delegated to Human
**Evidence:** The file `research/robert/science-questions.md` exists, but there is no evidence of an autonomous agent generating these questions. The `AGENTS.md` file frames agents as workers executing tasks on an established queue (`next-actions.md`), not as principal investigators defining new, unprompted research vectors.

## 2. Hypothesis Generation
**Status:** Partial
**Evidence:** The system tests multiple models (Boltzmann-Gibbs Blast-Wave, Tsallis, exact Bose-Einstein, Jüttner) as competing hypotheses. However, these models are hardcoded as Python functions within `physics/src/fitting_pipeline.py` (e.g., `blast_wave_fit_spec`, `tsallis_fit_spec`). The agents do not autonomously mathematically derive new models and implement them in code; they run predefined hypotheses against data.

## 3. Literature Synthesis
**Status:** Supported
**Evidence:** The repository has robust integrations for literature review.
*   `mcp_config.yaml` explicitly wires the environment to Consensus (for broad literature consensus), Scite (for citation context and contrasting/supporting metrics), and Onyx (for local document retrieval).
*   `physics/src/rag_claim_verifier.py` uses an LLM alongside the Onyx RAG index to classify claims as Supported, Contradicted, Nuanced, or Unsupported based on specific excerpts.

## 4. Experimental Design
**Status:** Partial
**Evidence:** The experimental design (fitting routines, multiplicity bins, parameter bounds) is largely predefined in Python scripts. However, there is evidence of advanced, reactive experimental testing, such as `run_fit_range_sensitivity.py`, which systematically truncates the dataset to test the stability of parameters (T_kin and beta_s). This shows sophisticated design, but it appears to be manually constructed by the human (Robert) rather than autonomously generated by the agent based on unexpected results.

## 5. Execution
**Status:** Supported
**Evidence:** The `physics/src/` directory contains fully functional, high-quality execution scripts. `fitting_pipeline.py` runs full Minuit optimizations, generates diagnostics, computes covariance matrices, and outputs residuals. `run_parallel_gpu.sh` indicates the system can scale execution.

## 6. Statistical Analysis
**Status:** Supported (Highly Rigorous)
**Evidence:** The statistical analysis capabilities are excellent for phenomenological physics.
*   **Goodness of fit:** Computes $\chi^2$/ndf, AIC, and BIC for model comparison (`fitting_pipeline.py`).
*   **Uncertainty & Degeneracy:** Generates and analyzes correlation matrices. `evidence-ledger.md` documents a critical finding where T_kin and $\beta_s$ were found to be statistically degenerate ($|\rho| > 0.95$), preventing their physical interpretation.
*   **Diagnostics:** Generates pull histograms overlaid with Normal distributions to check for systematic bias.

## 7. Visualization
**Status:** Supported
**Evidence:** `fitting_pipeline.py` contains `plot_fit_diagnostics()`, which generates 4-panel diagnostic figures (data+fit, residuals, pulls vs pT, pull histogram).

## 8. Peer Review & Replication
**Status:** Supported (Internal)
**Evidence:**
*   **Peer Review:** `research/robert/evidence-ledger.md` functions as a rigorous internal peer review gate. Claims cannot be promoted to "Validated" without overcoming explicit counter-arguments (e.g., the Bose-Einstein denominator investigation).
*   **Replication:** `AGENTS.md` mandates that all real runs are placed under `research/robert/runs/YYYY-MM-DD-*` with their configuration, preventing untrackable state.

## 9. Publication / Knowledge Accumulation
**Status:** Partial
**Evidence:** The `thesis/` and `manuscript/` directories indicate that the final goal is producing LaTeX documents. However, it is unclear how autonomously the system writes these. The knowledge accumulation into the `evidence-ledger.md` is excellent, acting as a structured, versioned graph of findings, but this is a pre-publication step.
