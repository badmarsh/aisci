# Tagging Taxonomy for Multica Knowledge Hub

Standardized labels ensure that our knowledge base remains searchable and organized.

## 1. Type Labels (Mandatory)
- `research`: Explorations, hypotheses, literature reviews.
- `implementation`: Features, code changes, deployments.
- `bug`: Root cause analysis, bug investigation notes.
- `decision`: Architectural choices, policy decisions.
- `integration`: Connecting systems, third-party services.
- `documentation`: Guides, READMEs, taxonomy updates.
- `workflow`: Results from automated workflows (DeerFlow).
- `idea`: Early stage thoughts not yet ready for research.

## 2. Domain Labels
- `physics`: General physics concepts.
- `blast-wave`: Specific to blast-wave models.
- `fitting`: Data fitting pipelines and algorithms.
- `infrastructure`: Servers, deployment, networking.
- `ui-ux`: Interface design and user experience.
- `data-science`: Data processing and analysis.

## 3. Source Labels
- `from-perplexity`: Context imported from Perplexity research.
- `from-claude-code`: Implementation notes from Claude Code sessions.
- `from-deerflow`: Automated summaries from DeerFlow runs.
- `from-onyx`: Insights from Onyx RAG queries.
- `from-slack`: Discussions moved from Slack for persistence.

## 4. Status and Priority
- Status: `backlog`, `todo`, `in_progress`, `in_review`, `done`, `blocked`, `cancelled`.
- Priority: `critical`, `high`, `medium`, `low`.
