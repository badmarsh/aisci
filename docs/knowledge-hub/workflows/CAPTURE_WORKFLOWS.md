# Knowledge Capture Workflows

Standardized ways to bring information into Multica from various tools.

## Perplexity Research
1. Export or copy the chat transcript from Perplexity.
2. Create a Multica issue using the `Research` template.
3. Use the `from-perplexity` label.
4. Extract key papers and findings into the structured sections.

## Claude Code Implementation
1. Review your session history for significant design choices or bug root causes.
2. Create an `Implementation` or `Bug` issue.
3. Use the `from-claude-code` label.
4. Document the rationale that won't be obvious from the code alone.

## DeerFlow Automations
1. Use the DeerFlow webhook to post a summary to Multica.
2. Ensure the summary includes the workflow name, results, and blockers.
3. Use the `workflow` and `from-deerflow` labels.

## Onyx RAG Queries
1. Capture significant RAG sessions where you found key literature.
2. Create a `Research` issue.
3. Use the `from-onyx` label.
